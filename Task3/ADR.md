### <a name="_3bfxc9a45514"></a>**Функциональные требования**

| № | Действующие лица или системы            | Use Case                    | Описание                                                                   |
|:-:|-----------------------------------------|-----------------------------|----------------------------------------------------------------------------|
| 1 | Клиент                                  | Подача заявки на депозит    | Клиент заполняет заявку через интернет-банк или сайт                       |
| 2 | Интернет-банк                           | Получение ставок            | Система запрашивает актуальные ставки для клиента                          |
| 3 | Интернет-банк,Интеграционный слой       | Передача заявки             | Интернет-банк отправляет заявку на депозит через прослойку                 |
| 4 | Интеграционный слой, АБС                | Обработка заявки            | Интеграционный слой передаёт заявку в АБС, получает результат              |
| 5 | АБС, Интеграционный слой, интернет-банк | Возврат результата          | Клиенту показывается результат заявки                                      |
| 6 | Кол-центр                               | Обработка заявки с сайта    | Кол-центр получает заявку с сайта, связывается с клиентом                  |
| 7 | Клиент, Служба поддержки                | Решение проблем             | Клиент обращается в поддержку при ошибках                                  |


### <a name="_u8xz25hbrgql"></a>**Нефункциональные требования**

- Система доступна 24/7, SLA ≥99.9%
- Обработка заявки ≤5 секунд
- Защищённое соединение HTTPS, шифрование
- Использование существующих технологий (.NET, MS SQL, Oracle, Delphi)
- Нет прямого подключения интернет-банка к АБС (обязательно через интеграционный слой)

### <a name="_bjrr7veeh80c"></a>**Альтернативы**

1. Разработка монолитного веб-приложения вместо микросервисной архитектуры
2. Интеграция напрямую с АБС без интеграционного слоя
3. Внешняя платформа как депозитный модуль (аутсорс)

**Недостатки, ограничения, риски**

- Требуется обновление интеграционного слоя для поддержки новой схемы
- Риск не консистентности ставок при медленном обновлении хранилища
- Устаревшие протоколы в АБС могут потребовать адаптеров
