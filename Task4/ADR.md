### <a name="_3bfxc9a45514"></a>**Функциональные требования**

| № | Участники                      | Use Case                  | Описание                                                                   |
| - | ------------------------------ | ------------------------- | -------------------------------------------------------------------------- |
| 1 | Сотрудник внутреннего КЦ       | Просмотр ставок через UI  | Получает актуальные ставки из системы депозитов через внутренний интерфейс |
| 2 | Сервис ставок → Система КЦ     | Передача ставок           | Сервис ставок предоставляет API для получения актуальных ставок            |
| 3 | Сотрудник партнёрского КЦ      | Загрузка ставок из файла  | Загружает ставки из CSV/XLSX-файла вручную                                 |
| 4 | Сервис ставок → Партнёрский КЦ | Ежедневная выгрузка файла | Формирует и отправляет файл ставок по защищённому каналу (например, SFTP)  |

### <a name="_u8xz25hbrgql"></a>**Нефункциональные требования**

| № | Требование                                                                     |
| - | ------------------------------------------------------------------------------ |
| 1 | Ставки должны обновляться в обеих КЦ не реже одного раза в сутки               |
| 2 | Для партнёрского КЦ использовать защищённый файловый обмен (SFTP + шифрование) |
| 3 | Отсутствие персональных данных в файлах                                        |
| 4 | Надёжность SLA ≥ 99.5%                                                         |
| 5 | Вся выгрузка логируется и подтверждается в журнале мониторинга                 |
| 6 | Поддержка ручной перезагрузки файла партнёром в случае сбоя                    |

### <a name="_bjrr7veeh80c"></a>**Альтернативы**
1. Прямое взаимодействие Интернет-банка с АБС (без отдельного API-шлюза)
2. Использование шины данных (ESB)
3. Использование микросервисов для оркестрации
4. Очереди сообщений (Message Queue)

**Недостатки, ограничения, риски**

- Возможна задержка при доставке файла в партнёрский КЦ.
- Необходимо обеспечить контроль формата и версии файла.
- Требуется реализовать безопасный канал для передачи файла.
- В случае сбоя выгрузки может быть показана устаревшая информация.
- Необходим аудит выгрузок и подтверждение успешной передачи.
